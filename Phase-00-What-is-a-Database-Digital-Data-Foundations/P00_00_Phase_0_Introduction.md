**Phase 0: Foundations of SQL for the Digital Library Project**

**Overview**

Welcome to Phase 0 of the Digital Library project, where we lay the groundwork for the SQL learning journey. In this phase, we'll introduce basic concepts of data, file systems, databases, and SQL, setting the stage for the development of a comprehensive catalog system for the university library.

**Phase Objectives**

By the end of Phase 0, you will be able to:

1. Understand the basics of data storage and retrieval
2. Design a file system to store book metadata and images
3. Set up a simple database management system (DBMS) using MySQL
4. Design a relational model to store book information, authors, and genres
5. Compare and contrast MySQL and PostgreSQL databases for the project

**Phase Tasks**

1. **Task 0.1: What is Data?**
	* Create a basic HTML page to display the library catalog
	* Understand the importance of data in the Digital Library project
2. **Task 0.2: How Computers Store Data**
	* Design a file system to store book metadata and images
	* Understand how computers store data using file systems
3. **Task 0.3: File Systems & How Databases Use Disk**
	* Implement a file system and database integration using disk storage
	* Understand how databases use disk storage to store data
4. **Task 0.4: What is a Database Management System (DBMS)?**
	* Set up a simple database management system (DBMS) using MySQL
	* Understand the basics of DBMS and its importance in the Digital Library project
5. **Task 0.5: Relational Model**
	* Design a relational model to store book information, authors, and genres
	* Understand the concept of relational databases and its application in the Digital Library project
6. **Task 0.6: SQL vs NoSQL — When to Use What**
	* Implement SQL queries to compare data storage and retrieval with NoSQL alternatives
	* Understand the differences between SQL and NoSQL databases and their use cases
7. **Task 0.7: MySQL vs PostgreSQL — Core Differences**
	* Compare and contrast MySQL and PostgreSQL databases for the project
	* Understand the key differences between these two popular databases
8. **Task 0.8: Dev Environment Setup**
	* Set up a development environment using code editors, terminals, and version control
	* Understand the importance of a well-configured development environment
9. **Task 0.9: Connecting to Databases**
	* Connect to the MySQL database using a JavaScript library and write SQL queries
	* Understand how to connect to a database using a programming language
10. **Task 0.10: How SQL Queries Execute (Conceptual)**
	* Visualize how SQL queries execute and optimize the database schema for better performance
	* Understand the conceptual aspects of SQL query execution and optimization

**Phase Deliverables**

1. A basic HTML page to display the library catalog
2. A designed file system to store book metadata and images
3. A simple database management system (DBMS) set up using MySQL
4. A relational model to store book information, authors, and genres
5. A comparison of SQL and NoSQL databases and their use cases
6. A comparison of MySQL and PostgreSQL databases for the project
7. A development environment set up using code editors, terminals, and version control
8. A connected MySQL database using a JavaScript library and written SQL queries
9. A visual representation of how SQL queries execute and an optimized database schema

**Phase Timeline**

This phase is designed to be completed within 2-3 weeks, with each task taking approximately 1-2 days to complete. The exact timeline may vary depending on your pace and the amount of time you dedicate to each task.

**Phase Grading**

Your progress will be evaluated based on the deliverables and the quality of your work. You will receive feedback on each task, and your overall performance will be assessed at the end of the phase.
